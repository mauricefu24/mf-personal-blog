## 一、项目背景与目标

### 1.1 背景

随着独立开发者内容创作需求的增长，市面上通用博客平台（如微信公众号、掘金、Medium）存在以下局限：

- 内容归属不清晰，平台随时可能限流或封号
- 排版与主题高度受限，无法体现个人品牌
- 无法聚合多元内容（阅读、烹饪、游戏等兴趣模块）
- 数据不可控，迁移成本高

**MF Personal Blog** 旨在打造一个完全自托管、可持续迭代的个人内容中心，既是对外展示工程实践能力的载体，也是个人知识沉淀与生活记录的平台。

### 1.2 目标

| 维度 | 目标 |
|------|------|
| 产品目标 | 打造多模块个人内容站，支持博客、阅读、烹饪、游戏等栏目 |
| 技术目标 | 验证 Next.js 16 + Prisma + SQLite 轻量全栈方案的可行性 |
| 运营目标 | 实现内容自主管理，无依赖第三方平台发布 |
| 体验目标 | 提供高质量暗色调 Premium UI，彰显独立开发者品牌调性 |

---

## 二、目标用户

### 2.1 用户画像

**主要用户：访客（内容消费者）**

- 技术同行：关注前端工程、数据库实践、自动化等技术内容
- 招聘方/合作伙伴：通过博客评估作者技术水平与表达能力
- 兴趣爱好者：对阅读书单、食谱、小游戏感兴趣的泛用户

**管理员用户：博主本人（内容生产者）**

- 需要便捷的内容发布工作流（草稿→发布）
- 需要管理文章分类、标签、个人资料

### 2.2 用户需求

| 用户类型 | 核心需求 |
|----------|----------|
| 访客 | 快速找到感兴趣的内容；阅读体验舒适；不被打扰 |
| 博主 | 高效写作与发布；内容可分类管理；多模块内容聚合 |

## 四、技术架构

### 4.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | Next.js 16 (App Router) | SSR + SSG 混合渲染，SEO 友好 |
| UI 样式 | Tailwind CSS 4 | 原子化 CSS，定制 Premium 暗色主题 |
| 语言 | TypeScript 5 | 全栈类型安全 |
| ORM | Prisma 7 | 数据库操作抽象层，支持迁移管理 |
| 数据库 | SQLite (better-sqlite3) | 零运维，适合个人博客体量 |
| 认证 | NextAuth 4 + bcryptjs | Cookie 会话，密码哈希 |
| 部署 | Docker + Docker Compose | 容器化，可部署至 VPS / 云服务器 |

## 五、非功能需求

| 类别 | 需求 |
|------|------|
| 性能 | 首屏 LCP < 2.5s（桌面端）；博客列表页使用 ISR 或 SSG |
| SEO | 每篇文章支持自定义 `seoTitle` / `seoDescription`；自动生成 Open Graph 标签 |
| 安全 | 管理员密码 bcrypt 哈希存储；后台路由服务端鉴权；防止 XSS（内容 sanitize）|
| 可维护性 | Prisma Migration 管理 schema 变更；Docker 一键部署；环境变量外部注入 |
| 可用性 | 移动端全响应式；核心内容无 JavaScript 也可访问（SSR） |
| 可观测性 | 服务端错误记录至控制台；后续可接入 Sentry / Axiom |
